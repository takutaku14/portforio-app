# **Universal Design Base Framework v1.0**

## **0\. はじめに (Introduction)**

本ドキュメントは、あらゆるデジタルプロダクト開発において、一貫性・保守性・アクセシビリティ・美しさを担保するための普遍的なデザイン基盤（Design Base）である。

プロジェクト開始時に本ガイドラインを参照し、具体的なブランド定義（色、フォント）を流し込むことで、デザインシステムを構築する。

## **1\. デザイン原則 (Core Principles)**

### **1.1 明瞭性 (Clarity)**

* **コンテンツファースト:** 装飾はコンテンツを際立たせるためにのみ存在する。  
* **不可視の階層:** 罫線（Border）に頼らず、余白（Spacing）と背景色（Surface）の違いで情報のグループを表現する。

### **1.2 フィードバック (Feedback)**

* **反応するUI:** ユーザーのすべてのアクション（クリック、タップ、ホバー）に対し、システムは必ず視覚的・触覚的な応答を返す。  
* **状態の可視化:** システムの状態（処理中、成功、エラー）は、ユーザーが探さなくても目に入る形で提示する。

### **1.3 普遍的なアクセシビリティ (Universal Accessibility)**

* **WCAG 2.1 AA準拠:** コントラスト比、キーボード操作、スクリーンリーダー対応を「追加機能」ではなく「要件」とする。  
* **タッチターゲット:** ポインティングデバイスを問わず、インタラクティブな要素は最低 44x44px の領域を持つ。

## **2\. 空間とレイアウト (Space & Layout)**

すべての配置は数学的な根拠に基づく。感覚的なピクセル調整を禁止する。

### **2.1 8pt グリッドシステム**

すべての余白（Margin, Padding）および要素のサイズは、**8の倍数**（微調整は4の倍数）で構成する。

| トークン名 | サイズ | 用途 |
| :---- | :---- | :---- |
| space-0.5 | 2px | 微細なボーダー調整 |
| space-1 | 4px | 要素内の最小余白、アイコンとテキストの間 |
| space-2 | 8px | 関連する要素間の距離（ラベルと入力欄など） |
| space-3 | 12px | space-2 と space-4 の中間が必要な場合 |
| space-4 | 16px | **\[基準\]** コンポーネント内の標準パディング |
| space-6 | 24px | コンポーネント間の距離 |
| space-8 | 32px | セクション間の距離 |
| space-12 | 48px | 大きなセクションの区切り |
| space-16 | 64px | 画面全体の余白（デスクトップ） |

### **2.2 ブレークポイント (Responsive)**

モバイルファーストで設計し、以下の基準点でレイアウトを変更する。

* **Mobile:** \< 640px (1カラム、パディング 16px)  
* **Tablet:** 640px \- 1024px (適宜2カラム、パディング 24px)  
* **Desktop:** \> 1024px (最大幅固定、中央寄せ、パディング 32px以上)

## **3\. カラーシステム (Color System)**

色は「見た目」ではなく「意味（Semantics）」で定義する。

RGB/Hexではなく、知覚的均一性を持つ OKLCH (または HSL) 色空間を推奨する。

### **3.1 セマンティックトークン (命名規則)**

ハードコード（例: \#3B82F6）を禁止し、以下の役割名を使用する。これによりダークモード対応が自動化される。

| カテゴリ | トークン名 | 役割 |
| :---- | :---- | :---- |
| **Background** | bg-base | アプリケーションの最背面 |
|  | bg-surface | カード、サイドバー、モーダルの背景 |
|  | bg-surface-elevated | ドロップダウンなど、浮いている要素の背景 |
| **Text** | text-main | 主要なテキスト（コントラスト比 7:1以上） |
|  | text-sub | 補足情報、メタデータ（コントラスト比 4.5:1以上） |
|  | text-placeholder | プレースホルダー、無効状態 |
| **Action** | primary | 主要アクション（ブランドカラー） |
|  | secondary | 補助アクション |
| **Status** | status-success | 成功、完了（緑系） |
|  | status-warning | 注意、警告（黄・橙系） |
|  | status-danger | 破壊的アクション、エラー（赤系） |

### **3.2 ダークモードの原則**

* **純黒の回避:** text-main に \#000000 を使用しない（\#111111 等のダークグレーを使用）。  
* **光源の逆転:** ダークモードでは「影」が見えないため、「背景色の明るさ」で奥行きを表現する（手前にあるものほど明るいグレーにする）。

## **4\. タイポグラフィ (Typography)**

可読性を最優先する。システムフォント（Inter, Roboto, SF Pro）を基本とし、装飾的なフォントは見出しのみに限定する。

### **4.1 タイプスケール**

基本サイズ（Base Size）を定め、比率（Ratio 1.2〜1.25）でスケールさせる。

| 役割 | サイズ (Desktop / Mobile) | Line Height | Tracking | Weight |
| :---- | :---- | :---- | :---- | :---- |
| **Display** | 48px / 36px | 1.1 | \-0.02em | Bold |
| **Heading 1** | 32px / 28px | 1.2 | \-0.01em | Bold |
| **Heading 2** | 24px / 22px | 1.3 | \-0.01em | SemiBold |
| **Heading 3** | 20px / 18px | 1.4 | 0 | SemiBold |
| **Body** | **16px** (or 17px) | 1.5 \- 1.6 | 0 | Regular |
| **Small** | 14px | 1.5 | 0 | Regular |
| **Caption** | 12px | 1.4 | \+0.02em | Medium |

### **4.2 文字組みのルール**

* **行長制限:** 本文の1行の文字数は、日本語で **35〜40文字**（英語で60〜80文字）程度に抑える。長すぎると視線移動が疲れる。  
* **行間 (Leading):** 見出しは狭く（1.1〜1.3）、本文は広く（1.5〜1.8）取る。  
* **字間 (Tracking):** 大きな文字は詰め（Tight）、小さな文字は広げる（Wide）。

## **5\. 形状とマテリアル (Shape & Material)**

### **5.1 角丸 (Radius)**

「親しみやすさ」と「規律」のバランスを定義する。

* **Small (4px):** チェックボックス、タグ、内包される要素。  
* **Medium (8px \- 12px):** ボタン、入力フィールド、標準的なカード。  
* **Large (16px \- 24px):** モーダル、大きなコンテナ。  
* **Full (9999px):** 丸ボタン、ステータスバッジ。

*注: プロジェクトの性格により、Sharp（0px）か Round（多め）かを統一する。混在させない。*

### **5.2 深度 (Elevation)**

Z軸（奥行き）を表現する方法。

* **Flat:** 背景に溶け込む。  
* **Raised:** わずかな影。操作可能であることを示唆（ボタン、カード）。  
* **Overlay:** 強い影。背景から分離（ドロップダウン、ヘッダー）。  
* **Modal:** 最も強い影と、背景のDimミング（暗転）。

## **6\. インタラクションとモーション (Interaction & Motion)**

### **6.1 アニメーションの原則**

* **即時性:** ユーザー操作に対するフィードバックは **100ms以内** に開始する。  
* **物理挙動:** ease-in-out よりも、バネの物理演算（Spring Physics）を優先する。  
* **継続性:** オブジェクトは唐突に消えず、フェードアウトや縮小を伴って退場する。

### **6.2 ステート（状態）定義**

インタラクティブな要素は以下の状態を必ずスタイル定義する。

1. **Default:** 通常時  
2. **Hover:** カーソルが乗った時（輝度を上げる、浮き上がる）  
3. **Active / Pressed:** 押下時（輝度を下げる、沈み込む）  
4. **Focus:** キーボード選択時（**アウトラインを必ず表示する**）  
5. **Disabled:** 無効時（不透明度を下げる、カーソルを禁止マークに）

## **7\. 実装・命名規則 (Implementation)**

デザインをコードに落とし込む際のルール。

### **7.1 CSS変数 / Design Tokens**

マジックナンバー（直接的な数値）を書かず、変数を参照する。

:root {  
  /\* 悪い例 \*/  
  \--header-height: 60px;   
  /\* 良い例（抽象化） \*/  
  \--size-header: var(--space-16);  
    
  \--color-primary: oklch(60% 0.2 250);  
  \--radius-md: 8px;  
}

### **7.2 Z-Index 戦略**

適当な数字（z-index: 9999）を使わず、役割ごとに定数を決める。

* z-10: フローティング要素（バッジ等）  
* z-100: 固定ヘッダー/フッター  
* z-200: ドロップダウン/ポップオーバー  
* z-300: モーダル/ダイアログ  
* z-400: トースト通知/ローディングオーバーレイ  
* z-500: 最前面のエラー表示

## **8\. チェックリスト (Quality Assurance)**

リリース前に必ず確認する項目。

* \[ \] **色のコントラスト:** テキストと背景のコントラスト比は十分か？  
* \[ \] **タップ領域:** スマホでタップしやすい大きさか（44px以上）？  
* \[ \] **ローディング:** 通信待ちの間にスケルトンやスピナーが表示されるか？  
* \[ \] **エラーハンドリング:** エラー時にユーザーが次に何をすべきか明示されているか？  
* \[ \] **レスポンシブ:** 画面幅を縮めた時にレイアウトが崩れないか？  
* \[ \] **拡大表示:** ブラウザの文字サイズを200%にしても破綻しないか？

### **このガイドラインの使い方**

1. **新規プロジェクトのルートに配置:** DESIGN\_BASE.md として保存してください。  
2. **プロジェクト固有の定義:**  
   * **3\. カラーシステム:** ブランドカラー（Primary）を決定し、OKLCH値を設定する。  
   * **4\. タイポグラフィ:** 使用するフォントファミリー（例: Noto Sans JP）を決定する。  
3. **開発チームへの共有:** 「迷ったらこれを見る」「これに書いてないスタイルは勝手に作らない」という合意形成を行う。
